# PCI DSS Requirements Summary

## Overview

The Payment Card Industry Data Security Standard (PCI DSS) protects payment card data.

## Requirement Structure

### 6 Goals and 12 Requirements

1. Install and maintain firewall configuration
2. Do not use vendor-supplied defaults
3. Protect stored cardholder data
4. Encrypt transmission of cardholder data
5. Protect systems against malware
6. Develop and maintain secure systems and applications
7. Restrict access by business need to know
8. Identify and authenticate access
9. Restrict physical access
10. Track and monitor network resources
11. Test security systems regularly
12. Maintain information security policy

## Key Strengths

- Industry-specific
- Detailed technical requirements
- Compliance verification
- Widely adopted

## Implementation Approach

1. Assess compliance
2. Identify gaps
3. Implement controls
4. Test and validate
5. Maintain compliance
