# GRC Engineering Labs - Study Guide

## Week-by-Week Learning Objectives

This study guide provides detailed learning objectives, key concepts, and preparation guidance for each week of the GRC Engineering Labs programme.

---

## Phase 1: Compliance & Governance Foundations

### Module GRC101: Introduction to GRC & Technical Landscape

#### Week 1: Linux for GRC Operations

**Learning Objectives:**
- Master essential Linux commands for GRC work
- Navigate file systems and manage permissions
- Use grep, awk, and sed for log analysis
- Understand Linux security basics

**Key Concepts:**
- File system hierarchy
- User and group management
- File permissions (rwx)
- Process management
- Log file locations

**Preparation:**
- Install Linux/WSL2
- Familiarize with terminal basics
- Review basic command syntax

**Assessment Focus:**
- Command proficiency
- Log analysis skills
- Security awareness

## Study Tips

### Time Management
- Allocate 5-8 hours per week per lab
- Start labs early in the week
- Don't procrastinate on difficult topics
- Plan time for documentation

### Learning Strategies
- Read lab instructions completely before starting
- Take notes on key concepts
- Practice commands and scripts
- Review framework documentation
- Ask questions early

### Documentation
- Document as you work
- Take screenshots immediately
- Keep organized notes
- Use the lab report template
- Cite sources properly

### Technical Skills
- Practice in safe environments
- Use virtual machines
- Keep backups of work
- Learn from errors
- Build a personal lab

### Framework Knowledge
- Read official framework documents
- Understand control objectives
- Practice mapping controls
- Study real-world implementations
- Stay current with updates

### Collaboration
- Discuss concepts with peers
- Join study groups
- Participate in discussions
- Share resources (not solutions)
- Help others learn

### Assessment Preparation
- Review rubrics before starting
- Self-assess using checklists
- Proofread all submissions
- Verify all requirements met
- Submit early when possible
