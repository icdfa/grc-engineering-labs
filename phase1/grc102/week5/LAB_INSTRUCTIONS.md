# Lab 5: Information Security Governance Fundamentals

**Course:** GRC102 - Information Security Governance  
**Week:** 5  
**Duration:** 5 days  
**Level:** Intermediate  
**Prerequisites:** GRC101 completion

## Lab Overview

This lab introduces information security governance principles, organizational structures, and roles. Participants will develop governance frameworks and understand board-level reporting.

## Learning Objectives

Upon completion of this lab, you will be able to:

- Understand information security governance principles
- Define roles and responsibilities in security governance
- Develop security governance frameworks
- Create board-level security reports
- Establish security steering committees

## Required Resources

- Linux system (Ubuntu 20.04 or later recommended)
- Terminal/SSH access
- Text editor (nano, vim, or VS Code)
- Lab report template (provided)
- Excel or LibreOffice Calc for data analysis
- Access to framework documentation (provided in resources/)
- Sample datasets (provided in datasets/)

## Lab Tasks

### Task 1: Security Governance Structure Design (Day 1)

**Objective:** Design organizational security governance structure with roles and responsibilities

**Instructions:**

1. Create a working directory for this task:
   ```bash
   mkdir -p ~/grc-lab-week5/task1
   cd ~/grc-lab-week5/task1
   ```

2. Review the task requirements and objectives carefully

3. Follow the step-by-step procedures below:

   **Step-by-Step Procedure:**
   
   a. Review policy templates and examples:
   ```bash
   ls ../../templates/
   cat ../../templates/policy_template.md
   ```
   
   b. Create policy document structure:
   ```bash
   cat > policy_document.md << 'EOF'
   # [Policy Title]
   
   ## 1. Purpose
   [State the purpose of this policy]
   
   ## 2. Scope
   [Define who and what this policy applies to]
   
   ## 3. Policy Statements
   [List specific policy requirements]
   
   ## 4. Roles and Responsibilities
   [Define who is responsible for what]
   
   ## 5. Compliance
   [Describe compliance requirements and consequences]
   
   ## 6. Review and Updates
   [Policy review schedule and update process]
   
   ---
   **Effective Date:** [Date]  
   **Review Date:** [Date]  
   **Approved By:** [Name/Title]
   EOF
   ```
   
   c. Develop policy content following best practices:
   - Use clear, concise language
   - Make requirements specific and measurable
   - Align with industry frameworks
   - Consider organizational context
   
   d. Create supporting procedures or standards as needed
   
   e. Document policy approval and implementation process
   
   f. Develop compliance monitoring approach
   

4. Document your findings and create deliverables

5. Review your work for completeness and accuracy

**Deliverable:** Completed task documentation, analysis, and supporting files
